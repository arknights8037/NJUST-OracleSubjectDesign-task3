## 一、功能需求

### 1. 人事档案管理
- 员工入职时建立人事档案，档案由**公司经理级及以上人员**维护。
- 档案包含：入职时间、员工级别、基本工资、所在部门、姓名、性别、身份证号、联系方式等基本信息。
- 支持员工调动和离职。**调动和离职必须保留完整历史信息**（原部门、原级别、原工资、操作时间等），可随时查询。

### 2. 工资管理
- 工资 = 基本工资 + 奖金。
- **基本工资**由员工级别决定，级别分为：试用员工、正式员工、组长、部门经理、总经理。每个级别对应固定基本工资金额，此对应关系应可配置。
- **奖金**由员工的直接领导或更高级领导根据业绩确定，支持录入和修改。
- 系统需能生成月度工资单，并支持批量处理。

### 3. 考勤管理
- 员工上班时**签到**，下班时**签离**，系统记录精确时间。
- 因公外出或出差，需由**上级领导确认**考勤状态（正常、出差、外出）。
- 缺勤或迟到需**自动计算罚款**：迟到 1 小时以上视为缺勤；缺勤和迟到的罚款金额可在基础数据中维护。
- 考勤记录可供员工本人及管理人员查询。

### 4. 基础数据维护
系统应提供界面维护以下基础数据：
- 部门信息
- 员工级别及对应的基本工资标准
- 迟到/缺勤罚款金额
- 其他可配置参数

### 5. 统计查询与报表
- 强大的数据统计功能：如各部门人数统计、工资总额统计、月考勤汇总等。
- 综合查询：按姓名、部门、级别、入职时间范围等条件组合查询员工信息（含历史记录）。
- 报表生成与打印：月度工资报表、员工花名册、考勤统计报表等，支持前端直接打印或导出 Excel/PDF。

### 6. 用户与权限
- 系统登录使用 **JWT 认证**。
- 角色至少包含：普通员工、部门经理、总经理（或系统管理员）。
- 经理及以上可管理人事档案、审批考勤、设定奖金等；普通员工只能查看个人信息、签到签离、查看个人工资等。

### 7. 非功能性需求
- **图形用户界面**使用 Element Plus 构建，美观易用，导航清晰。
- 服务端可运行在 Windows 或 UNIX 平台，客户端运行于 Windows，需保证良好的跨平台特性。
- 系统需具备**良好的可扩展性**，代码结构清晰，模块化设计。
- 所有业务接口遵循 RESTful 风格。
- 配置文件使用toml
